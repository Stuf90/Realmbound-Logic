# ROYAL INQUEST — RULES

> AGENT FILE. CAVE SPEAK. HUMAN VERSION: `rules.human.md`.

ROYAL INQUEST = SPATIAL MURDOKU-STYLE PLACEMENT PUZZLE. PLACE EVERY CHARACTER ON BOARD OF
CHAMBERS SO ALL CLUES HOLD SAME TIME. THIS DOC = CANONICAL **GAME RULES** — HOW FINISH
CASE PLAY + JUDGE. NOT AUTHOR NEW CONTENT — THAT IN SUB-DOCS BOTTOM.

## BOARD

- GRID `rows x columns` CELLS (`InquestDefinition.rows/columns`).
- EVERY CELL BELONG ONE **CHAMBER** (`chamberId`) — NAME ROOM, VISIBLE BOUNDARY + LABEL.
  CHAMBERS GROUP CELLS, NOT SEPARATE GRID.
- SOME CELLS **BLOCKED** — IMPASSABLE. NO CHARACTER EVER OCCUPY. BLOCKED CELL MAY HOLD
  **DECORATIVE** PROP (BOOKSHELF, DUNGEON CAGE, ...).
- EVERY UNBLOCKED CELL LEGAL FOR **ANY** CHARACTER — NO PER-CELL CHARACTER RESTRICT.
  UNBLOCKED CELL MAY HOLD **SEAT** PROP (CHAIR, BENCH, PEW, THRONE); CHARACTER PLACE
  THERE EXACT LIKE ANY OTHER UNBLOCKED CELL, PROP RENDER UNDER PORTRAIT.

## CHARACTERS

- CASE HAVE FIXED CAST, EACH NAME + PORTRAIT.
- EXACTLY ONE = **VICTIM**.
- EXACTLY ONE = **TRAITOR**, DECIDE BY WHERE EVERYONE END UP (SEE "VICTIM + TRAITOR"
  BELOW) — NOT AUTHOR FACT PLAYER PLACE DIRECT.
- NO CLUE EVER NAME VICTIM DIRECT. VICTIM CELL ONLY REACH BY ELIMINATION, FROM CLUES
  ABOUT EVERYONE ELSE.

## PLACEMENT RULES

FINISH PLACEMENT VALID ONLY WHEN **ALL** HOLD:

1. **ONE CHARACTER PER ROW.** NO TWO SHARE ROW.
2. **ONE CHARACTER PER COLUMN.** NO TWO SHARE COLUMN.
3. **UNBLOCKED CELL ONLY.** CHARACTER USE UNBLOCKED CELL ONLY.
4. **EVERY CLUE SATISFIED.** SEE "CLUES" BELOW.
5. **VICTIM/TRAITOR CONDITION HOLD** (SEE BELOW).

ROWS/COLUMNS INCREASE TOP-TO-BOTTOM, LEFT-TO-RIGHT. 0-INDEX INTERNAL, 1-INDEX IN CLUE
TEXT (ROW INDEX `0` = "FIRST ROW").

## CHAMBERS + ADJACENCY

- **SAME CHAMBER** — TWO CHARACTERS SAME `chamberId`.
- **DIFFERENT CHAMBER** — OPPOSITE.
- **BESIDE** — TWO CHARACTERS ORTHOGONAL ADJACENT (MANHATTAN DIST 1) **+** SAME CHAMBER.
  ADJACENT ACROSS CHAMBER BOUNDARY NOT COUNT.
- **NOT BESIDE** — NEGATE "BESIDE": ANY PLACEMENT NOT BOTH ADJACENT + SAME-CHAMBER.

## CLUES

EVERY CLUE = STRUCTURED PREDICATE, NOT FREE TEXT — SENTENCE SHOW PLAYER = FLAVOR ONLY,
NEVER PARSE FOR CORRECTNESS. SEE
[`authoring/clues-and-predicates.cave.md`](authoring/clues-and-predicates.cave.md) FULL
PREDICATE LIST, WHICH PREDICATE CLUE ACTUAL ALLOW USE, + EXACT SEMANTICS EACH.

CLUE STATE WHILE PLACE STILL GO:

- **SATISFIED** — TRUE GIVEN CURRENT (MAYBE PARTIAL) PLACEMENT.
- **VIOLATED** — DEFINITE FALSE ALREADY, NO MATTER WHAT UNPLACED STILL.
- **UNDETERMINED** — CAN'T DECIDE YET, NEED CHARACTER NOT PLACED.

ONLY DEFINITE VIOLATION REPORT BEFORE COMPLETE — PLAYER NEVER TOLD WRONG WHEN MERELY
STILL UNKNOWN.

## VICTIM + TRAITOR

TRAITOR NOT INDEPENDENT PLACEMENT FACT — FALL OUT FROM EVERYONE ELSE POSITION:

> TRAITOR = ONLY NON-VICTIM CHARACTER SHARE VICTIM'S CHAMBER, WHEN NO THIRD CHARACTER
> ALSO PRESENT THERE.

MEAN: AT COMPLETE, VICTIM CHAMBER MUST HOLD EXACTLY TWO SOLVE OCCUPANT — VICTIM +
TRAITOR — NO ONE ELSE. ALSO ENFORCE AUTHOR TIME BY ELIMINATION: ONCE EVERY CHARACTER
EXCEPT VICTIM PLACED, EXACTLY ONE UNBLOCKED CELL MUST STAY OPEN TO VICTIM, ITS CHAMBER
ONLY OTHER OCCUPANT MUST BE TRAITOR. SEE
[CHARACTER PLACEMENT](authoring/character-placement.cave.md).

## COMPLETION

CASE COMPLETE ONLY WHEN:

- ALL CHARACTERS LEGAL PLACED (RULE 3);
- ROW + COLUMN UNIQUE HOLD (RULE 1-2);
- EVERY CLUE SATISFIED (RULE 4);
- VICTIM/TRAITOR CONDITION HOLD (RULE 5);
- EVERY CHARACTER SIT ON PUZZLE'S AUTHOR SOLUTION CELL.

LAST POINT MATTER EVEN THOUGH STRUCTURAL RULES ABOVE CHECK INDEPENDENT — CASE HAVE
EXACTLY ONE AUTHOR SOLUTION, COMPLETE NEED REACH IT, NOT MERELY SATISFY EVERY RULE
ABSTRACT (PUZZLE AUTHOR SO THOSE COINCIDE, AND AUTHOR ITSELF VERIFY EXACTLY ONE VALID
SOLUTION — SEE "MAKE NEW CONTENT" BELOW).

## PLAYER TOOLS

- **PLACE** — PUT SELECT CHARACTER ON UNBLOCKED CELL. COMMIT THERE — PLACE OVER TILE
  REMOVE ANY DRAFT NOTE ON THAT SPECIFIC TILE FOR THAT CHARACTER (SEE NOTE/DRAFT BELOW),
  COUNT FOR REAL AGAINST CLUE + ROW/COLUMN UNIQUE. PLACE INTO ROW/COLUMN ANOTHER
  CHARACTER ALREADY OCCUPY REJECT + FLASH ATTEMPT CELL RED BRIEF.
- **NOTE/DRAFT** — MARK CELL AS POSSIBLE CANDIDATE FOR SELECT CHARACTER, NO COMMIT.
  UNLIKE PLACE, SAME CHARACTER CAN DRAFT ONTO MULTIPLE CELL SAME TIME — SCRATCHPAD OF
  "MAYBE HERE" GUESS. EACH DRAFT CELL SHOW CHARACTER FIRST-NAME INITIAL, SMALLER THAN
  AVATAR/PROP ART. NOTE + PLACE SHARE ONE TOOLBAR SLOT: NOT DRAFTING → BUTTON READ
  "NOTE", START DRAFT; DRAFTING → BUTTON READ "PLACE", RETURN PLACE MODE.
- **CROSS** — MANUAL MARK CELL IMPOSSIBLE, PLAYER OWN BOOKKEEP. INDEPENDENT FROM GAME
  OWN AUTO-CROSS CELL (CELLS GAME ALREADY MARK UNAVAILABLE BECAUSE ANOTHER PLACE
  CHARACTER SHARE ROW/COLUMN) — NEVER AUTO-CLEAR WHEN AUTO-CROSS CHANGE. CLICK MARK CELL
  AGAIN REMOVE CROSS — EXCEPT MANUAL CROSS CAN'T REMOVE WHILE ROW/COLUMN STILL HOLD
  PLACE CHARACTER, SINCE GAME OWN AUTO-CROSS WOULD IMMEDIATE RE-MARK ANYWAY.
- **HINT** — REPORT EXIST CONTRADICTION FIRST (ILLEGAL PLACEMENT, DUPLICATE ROW,
  DUPLICATE COLUMN, VIOLATE CLUE, CHARACTER NO LEGAL CELL LEFT, INVALID VICTIM/TRAITOR
  CHAMBER); ELSE OFFER ONE DETERMINISTIC DEDUCTION + REASON, OPTION APPLY.
- **RESET** — ERASE ALL PLACEMENT, DRAFT, CROSS — START CASE OVER, AFTER CONFIRM.
